## 🌐 国际主流平台

### 1. Vercel ⭐⭐⭐⭐⭐

<details>
<summary><strong>平台特点</strong></summary>

**优势**：
- 🚀 **部署速度极快** - 全球CDN加速
- ✨ **开发体验最佳** - 零配置部署
- 🔄 **自动部署** - Git推送自动触发
- 📊 **性能分析** - 内置Web Vitals监控
- 🔒 **自动HTTPS** - 免费SSL证书
- 🎯 **预览部署** - 每个PR自动生成预览链接

**免费额度**：
- 100GB 带宽/月
- 无限网站和部署
- 自动HTTPS
- 自定义域名

**适用场景**：
- ✅ 前端项目展示
- ✅ 个人博客
- ✅ 产品Demo
- ✅ React/Vue/Next.js项目

</details>

<details>
<summary><strong>快速开始</strong></summary>

**方法1：通过GitHub导入（推荐）**

```bash
# 1. 推送代码到GitHub
git push origin main

# 2. 访问 https://vercel.com
# 3. 点击 "Import Project"
# 4. 选择你的GitHub仓库
# 5. 点击 "Deploy" - 完成！
```

**方法2：使用CLI**

```bash
# 安装Vercel CLI
npm i -g vercel

# 在项目目录运行
vercel

# 按提示完成配置，自动部署
```

**自定义域名**：
1. Vercel控制台 → Settings → Domains
2. 添加你的域名
3. 配置DNS：`CNAME` → `cname.vercel-dns.com`

</details>

<details>
<summary><strong>配置文件（可选）</strong></summary>

创建 `vercel.json`：

```json
{
  "buildCommand": null,
  "outputDirectory": ".",
  "framework": null,
  "redirects": [
    {
      "source": "/",
      "destination": "/web/",
      "permanent": false
    }
  ]
}
```

</details>

### 2. Netlify ⭐⭐⭐⭐⭐

<details>
<summary><strong>平台特点</strong></summary>

**优势**：
- 🎨 **拖拽部署** - 最简单的部署方式
- 🔄 **持续部署** - Git集成自动部署
- 📋 **表单处理** - 内置表单后端
- 🎯 **A/B测试** - 分流测试功能
- 🔀 **分支预览** - 每个分支独立URL
- 📦 **构建插件** - 丰富的插件生态

**免费额度**：
- 100GB 带宽/月
- 300分钟构建时间/月
- 自动HTTPS
- 自定义域名
- 表单提交（100次/月）

**适用场景**：
- ✅ 个人项目
- ✅ 小型商业网站
- ✅ 静态博客
- ✅ 需要表单功能的网站

</details>

<details>
<summary><strong>快速开始</strong></summary>

**方法1：拖拽部署（最简单）**

1. 访问 https://app.netlify.com/drop
2. 将项目文件夹拖入浏览器
3. 完成！获得访问链接

**方法2：Git集成**

```bash
# 1. 推送代码到GitHub/GitLab/Bitbucket
git push origin main

# 2. 访问 https://app.netlify.com
# 3. 点击 "New site from Git"
# 4. 选择仓库
# 5. 配置构建设置：
#    - Build command: (留空)
#    - Publish directory: .
# 6. 点击 "Deploy site"
```

**方法3：使用CLI**

```bash
# 安装Netlify CLI
npm install -g netlify-cli

# 登录
netlify login

# 初始化并部署
netlify init
netlify deploy --prod
```

</details>

<details>
<summary><strong>配置文件</strong></summary>

创建 `netlify.toml`：

```toml
[build]
  publish = "."
  command = ""

[[redirects]]
  from = "/"
  to = "/web/"
  status = 302

[[redirects]]
  from = "/*"
  to = "/index.html"
  status = 200
```

</details>

- 🔒 **DDoS防护** - 内置安全防护
- 🆓 **完全免费** - 个人使用零成本

- 500次构建/月
- 并发构建：1个

**适用场景**：
- ✅ 高流量网站
- ✅ 全球访问项目
- ✅ 需要DDoS防护
- ✅ 长期运营项目

<details>
<summary><strong>快速开始</strong></summary>

**通过Git部署**

```bash
# 1. 推送代码到GitHub/GitLab
git push origin main

# 2. 访问 https://pages.cloudflare.com
# 3. 点击 "Create a project"
# 4. 连接Git账号
# 5. 选择仓库
# 6. 配置：
#    - Build command: (留空)
#    - Build output directory: /
# 7. 点击 "Save and Deploy"
```

**使用Wrangler CLI**

```bash
# 安装Wrangler
npm install -g wrangler

# 登录
wrangler login

# 部署
wrangler pages publish . --project-name=my-project
```

<details>
<summary><strong>平台特点</strong></summary>

**优势**：

**适用场景**：
- ✅ 开源项目文档
- ✅ 个人博客
- ✅ 项目展示页
- ✅ 技术文档

</details>

<details>
<summary><strong>快速开始</strong></summary>

**方法1：通过仓库设置**

```bash
# 1. 推送代码到GitHub
git push origin main

# 2. GitHub仓库页面：Settings → Pages
# 3. Source选择: main分支
# 4. 文件夹选择: / (root)
# 5. 点击Save
# 6. 等待几分钟，访问：
#    https://USERNAME.github.io/REPO_NAME/
```

**方法2：使用gh-pages分支**

```bash
# 安装gh-pages工具
npm install -g gh-pages

# 部署到gh-pages分支
gh-pages -d .
```

**自定义域名**：
1. 在根目录创建 `CNAME` 文件，内容为域名
2. DNS配置：`CNAME` → `USERNAME.github.io`

</details>

- 360MB/天 传输（约10GB/月）
- 自定义域名
- SSL证书

**适用场景**：
- ✅ 需要后端服务的项目
- ✅ 需要实时数据库
- ✅ 移动应用配套网站

</details>

<details>
<summary><strong>快速开始</strong></summary>

```bash
# 1. 安装Firebase CLI
npm install -g firebase-tools

# 2. 登录Firebase
firebase login

# 3. 初始化项目
firebase init hosting

# 4. 配置（选择公共目录等）

# 5. 部署
firebase deploy
```

**配置文件 `firebase.json`**：

```json
{
  "hosting": {
    "public": ".",
    "ignore": [
      "firebase.json",
      "**/.*",
      "**/node_modules/**"
    ],
    "rewrites": [
      {
        "source": "/",
        "destination": "/web/index.html"
      }
    ]
  }
}
```

</details>

- 流量：0.50元/GB（CDN加速）
- 新用户有免费额度

**适用场景**：
- ✅ 商业项目
- ✅ 高流量网站
- ✅ 需要备案的网站
- ✅ 企业官网

</details>

<details>
<summary><strong>快速开始</strong></summary>

**1. 创建COS存储桶**

```bash
# 腾讯云控制台 → 对象存储 → 创建存储桶
# 配置：
#   - 访问权限：公有读私有写
#   - 静态网站：开启
#   - 索引文档：index.html
```

**2. 使用CLI上传**

```bash
# 安装COSCMD
pip install coscmd

# 配置
coscmd config -a <SecretId> -s <SecretKey> -b <BucketName> -r <Region>

# 上传文件
coscmd upload -r . /

# 或使用Web控制台直接上传
```

**3. 配置CDN加速**

```bash
# 腾讯云控制台 → CDN → 添加域名
# 源站类型：对象存储COS
# 选择刚创建的存储桶
```

**4. 访问网站**

```
http://your-bucket.cos-website.ap-guangzhou.myqcloud.com
或使用绑定的域名
```

</details>

./ossutil64 config

# 上传
./ossutil64 cp -r . oss://your-bucket/
```

```
http://your-bucket.oss-cn-hangzhou.aliyuncs.com/index.html

## 🔧 各平台配置文件参考

### Vercel (`vercel.json`)

```json
{
  "buildCommand": null,
  "outputDirectory": ".",
  "framework": null,
  "redirects": [
    {
      "source": "/",
      "destination": "/web/",
      "permanent": false
    }
  ],
  "headers": [
    {
      "source": "/(.*)",
      "headers": [
        {
          "key": "Cache-Control",
          "value": "public, max-age=3600"
        }
      ]
    }
  ]
}
```

### Netlify (`netlify.toml`)

```toml
[build]
  publish = "."
  command = ""

[[redirects]]
  from = "/"
  to = "/web/"
  status = 302

[[redirects]]
  from = "/*"
  to = "/index.html"
  status = 200

[[headers]]
  for = "/*"
  [headers.values]
    Cache-Control = "public, max-age=3600"
    X-Frame-Options = "DENY"
    X-Content-Type-Options = "nosniff"
```

### Cloudflare Pages (`_redirects`)

```
/  /web/  302
/*  /index.html  200
```

### GitHub Pages (`CNAME`)

```
yourdomain.com
```

## 📋 快速决策表

### 我应该选择哪个平台？

回答以下问题：

1. **目标用户在哪里？**
   - 主要在中国 → Gitee Pages / 腾讯云COS
   - 主要在海外 → Vercel / Netlify
   - 全球用户 → Cloudflare Pages

2. **预算如何？**
   - 完全免费 → Vercel / Netlify / Cloudflare
   - 可承受少量费用 → 腾讯云/阿里云（更稳定）

3. **技术水平？**
   - 新手 → Netlify（拖拽部署）
   - 熟悉Git → GitHub Pages / Gitee Pages
   - 专业开发 → Vercel / Cloudflare

4. **是否需要自动部署？**
   - 需要 → Vercel / Netlify / Cloudflare
   - 不需要 → Gitee Pages 也可以

5. **预期流量？**
   - <10GB/月 → 任何平台都可以
   - 10-100GB/月 → Vercel / Netlify / Cloudflare
   - >100GB/月 → Cloudflare Pages（无限）/ 云服务商CDN
