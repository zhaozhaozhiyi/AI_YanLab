## 🔧 路径方案调整说明（v2.3 更新）

### 核心变更
**从 Router.navigate() 转向 a标签 + 相对路径**

### 调整原因
1. ✅ **SEO友好** - 支持右键打开、搜索引擎爬取
2. ✅ **更简单** - 零JS依赖，原生HTML导航
3. ✅ **性能更好** - 无JS计算开销
4. ✅ **可维护性** - 路径可见，易于调试
5. ✅ **符合实际** - 项目代码主要已使用此方案

### 文档调整
1. **保留并完善** - `相对路径快速参考.md`（当前推荐方案）
2. **归档废弃** - `路径兼容性方案说明.md` → `_archive/路径兼容性方案说明-已废弃.md`
3. **明确指引** - 在README中标注当前推荐方案

### Router.js 保留用途
虽然不再推荐用于页面跳转，但 `shared/router.js` 仍保留以下用途：
- ✅ 登录状态检查：`Router.checkLogin()`
- ✅ URL参数获取：`Router.getQuery()`
- ✅ 环境检测：`Router.getEnvironmentInfo()`
- ✅ 退出登录：`Router.logout()`

详见：[相对路径快速参考.md - Router.navigate() 使用场景](./相对路径快速参考.md#-routernavigate-使用场景)
